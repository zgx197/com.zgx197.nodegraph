## 📌 先看这一点

NodeGraph 当前主架构已经不是早期的 `IDrawContext` 直绘模式，而是：

1. 纯 C# 层构建 `GraphFrame`
2. 引擎适配层消费 `GraphFrame`
3. 业务层通过目录、策略和渲染器接口注入扩展能力

也就是说，现在最值得关注的接口不是“怎么画矩形”，而是：

- `INodeTypeCatalog`
- `IConnectionPolicy`
- `IConnectionValidator`
- `INodeContentRenderer`
- `IEdgeLabelRenderer`
- `IPlatformInput`
- `IEditContext`
- `ITextMeasurer`
- `IGraphSerializer`
- `IGraphPersistence`

## 🔧 核心模型相关接口

<a id="core"></a>
### `INodeData` / `IEdgeData`

业务层数据的标记接口，由业务层自行实现并挂到 `Node.UserData` / `Edge.UserData` 上。

```csharp
public interface INodeData { }
public interface IEdgeData { }
```

适用场景：

- 节点配置数据
- 连线上的业务条件或元数据
- 业务层自己的运行时引用信息

### `INodeTypeCatalog`

节点类型目录接口。`GraphSettings` 和 `BlueprintProfile` 都依赖它来查找类型定义、驱动搜索菜单和创建节点默认端口。

```csharp
public interface INodeTypeCatalog
{
    NodeTypeDefinition? GetNodeType(string typeId);
    IEnumerable<NodeTypeDefinition> GetAll();
    IEnumerable<NodeTypeDefinition> Search(string keyword);
    IEnumerable<string> GetCategories();
}
```

说明：

- 内置有 `NodeTypeRegistry`，但它当前是框架内部实现，不建议把对外文档建立在直接 `new NodeTypeRegistry()` 上
- 业务层更推荐提供自己的目录实现，或通过工厂返回一个封装好的目录对象

### `IConnectionPolicy`

连接策略接口，决定两个端口能否建立连接。

```csharp
public interface IConnectionPolicy
{
    ConnectionResult CanConnect(Graph graph, Port source, Port target);
}
```

相关枚举 `ConnectionResult` 包含：

- `Success`
- `SameNode`
- `SameDirection`
- `KindMismatch`
- `DataTypeMismatch`
- `CapacityExceeded`
- `CycleDetected`
- `DuplicateEdge`
- `CustomRejected`

通常做法：

- 直接使用 `DefaultConnectionPolicy`
- 或继承 `DefaultConnectionPolicy` 后叠加业务规则

### `IConnectionValidator`

连接校验责任链中的单个验证节点。适合在不重写整套连接策略的前提下，为特定业务追加规则。

```csharp
public interface IConnectionValidator
{
    ConnectionResult? Validate(Graph graph, Port outPort, Port inPort);
}
```

返回值约定：

- 返回 `null`：本校验器放行，继续走后续规则
- 返回非空 `ConnectionResult`：拒绝连接，并以该结果作为最终原因

### 示例：自定义连接策略

```csharp
public sealed class MyConnectionPolicy : DefaultConnectionPolicy
{
    public override ConnectionResult CanConnect(Graph graph, Port source, Port target)
    {
        var result = base.CanConnect(graph, source, target);
        if (result != ConnectionResult.Success)
            return result;

        var sourceNode = graph.FindNode(source.NodeId);
        var targetNode = graph.FindNode(target.NodeId);

        if (sourceNode?.TypeId == "Start" && targetNode?.TypeId == "Start")
            return ConnectionResult.CustomRejected;

        return ConnectionResult.Success;
    }
}
```

## 🖼️ View 与渲染相关接口

<a id="view"></a>
### `IPlatformInput`

平台输入抽象接口。宿主窗口每帧把引擎事件转成该接口可读取的状态，`GraphViewModel.ProcessInput()` 再消费它。

```csharp
public interface IPlatformInput
{
    Vec2 MousePosition { get; }
    Vec2 MouseDelta { get; }
    float ScrollDelta { get; }

    bool IsMouseDown(MouseButton button);
    bool IsMouseUp(MouseButton button);
    bool IsMouseDrag(MouseButton button);
    bool IsDoubleClick(MouseButton button);

    bool IsKeyDown(string keyName);
    bool IsKeyHeld(string keyName);

    bool IsShiftHeld { get; }
    bool IsCtrlHeld { get; }
    bool IsAltHeld { get; }

    string GetClipboardText();
    void SetClipboardText(string text);
}
```

Unity 对应实现：`NodeGraph.Unity.UnityPlatformInput`

### `IEditContext`

节点内嵌编辑器上下文，提供 IMGUI 风格控件接口。

```csharp
public interface IEditContext
{
    void Begin(Rect2 availableRect);

    float FloatField(string label, float value);
    int IntField(string label, int value);
    string TextField(string label, string value);
    bool Toggle(string label, bool value);
    float Slider(string label, float value, float min, float max);
    int Popup(string label, int selectedIndex, string[] options);
    Color4 ColorField(string label, Color4 value);

    void Label(string text);
    void Space(float pixels);
    void BeginHorizontal();
    void EndHorizontal();
    bool Foldout(string label, bool expanded);
    void Separator();

    bool HasChanged { get; }
    Rect2 AvailableRect { get; }
}
```

Unity 对应实现：`NodeGraph.Unity.UnityEditContext`

### `ITextMeasurer`

文字测量接口，供 `FrameBuilder` 在纯 C# 层计算节点尺寸时使用。

```csharp
public interface ITextMeasurer
{
    Vec2 MeasureText(string text, float fontSize);
}
```

Unity 对应实现：`NodeGraph.Unity.UnityTextMeasurer`

### `INodeContentRenderer`

节点内容区渲染器接口。业务层通过它控制“节点内部显示什么”，框架层只负责标题栏、端口、边框和外层布局。

```csharp
public interface INodeContentRenderer
{
    bool SupportsInlineEdit { get; }

    Vec2 GetSummarySize(Node node, ITextMeasurer measurer);
    NodeContentInfo GetSummaryInfo(Node node, Rect2 contentRect);
    string GetOneLiner(Node node);

    Vec2 GetEditorSize(Node node, IEditContext ctx);
    void DrawEditor(Node node, Rect2 rect, IEditContext ctx);
}
```

与旧版不同的点：

- 不再使用 `IDrawContext` 直接绘制摘要
- 摘要模式返回的是 `NodeContentInfo` 数据，由引擎渲染层消费
- 编辑模式仍由引擎层直接调用 `DrawEditor()`

### 示例：节点内容渲染器

```csharp
public sealed class SpawnTaskContentRenderer : INodeContentRenderer
{
    public bool SupportsInlineEdit => true;

    public Vec2 GetSummarySize(Node node, ITextMeasurer measurer)
    {
        return new Vec2(180f, 44f);
    }

    public NodeContentInfo GetSummaryInfo(Node node, Rect2 contentRect)
    {
        var data = (SpawnTaskData)node.UserData!;

        var info = new NodeContentInfo
        {
            ContentRect = contentRect
        };

        info.SummaryLines.Add($"怪物: {data.TemplateName}");
        info.SummaryLines.Add($"波次: {data.WaveCount} / 间隔: {data.Interval:0.##}s");
        return info;
    }

    public string GetOneLiner(Node node)
    {
        var data = (SpawnTaskData)node.UserData!;
        return $"{data.TemplateName} / {data.WaveCount}波";
    }

    public Vec2 GetEditorSize(Node node, IEditContext ctx) => new Vec2(220f, 100f);

    public void DrawEditor(Node node, Rect2 rect, IEditContext ctx)
    {
        var data = (SpawnTaskData)node.UserData!;
        data.TemplateName = ctx.TextField("模板", data.TemplateName);
        data.WaveCount = ctx.IntField("波次", data.WaveCount);
        data.Interval = ctx.FloatField("间隔", data.Interval);
    }
}
```

### `IEdgeLabelRenderer`

连线标签渲染器接口。用于为边生成标签数据，并处理标签点击。

```csharp
public interface IEdgeLabelRenderer
{
    Vec2 GetLabelSize(Edge edge, ITextMeasurer measurer);
    EdgeLabelInfo GetLabelInfo(Edge edge, Vec2 midpoint);
    bool HandleLabelClick(Edge edge, Rect2 labelRect);
}
```

与节点内容渲染器类似，这里也已经是“返回数据，由渲染器消费”的思路。

### 示例：边标签渲染器

```csharp
public sealed class TransitionLabelRenderer : IEdgeLabelRenderer
{
    public Vec2 GetLabelSize(Edge edge, ITextMeasurer measurer)
    {
        string text = GetText(edge);
        var size = measurer.MeasureText(text, 10f);
        return size + new Vec2(12f, 6f);
    }

    public EdgeLabelInfo GetLabelInfo(Edge edge, Vec2 midpoint)
    {
        return new EdgeLabelInfo
        {
            Edge = edge,
            Position = midpoint,
            Text = GetText(edge),
            Size = new Vec2(72f, 18f)
        };
    }

    public bool HandleLabelClick(Edge edge, Rect2 labelRect)
    {
        return false;
    }

    private static string GetText(Edge edge) => edge.UserData?.ToString() ?? "";
}
```

## 💾 序列化与持久化接口

<a id="serialization"></a>
### `IGraphSerializer`

图序列化接口，负责把内存中的图转成中间格式，并支持子图复制粘贴场景。

```csharp
public interface IGraphSerializer
{
    string Serialize(Graph graph);
    Graph? Deserialize(string data);
    string SerializeSubGraph(Graph graph, IEnumerable<string> nodeIds);
    IEnumerable<Node> DeserializeSubGraphInto(Graph target, string data, Vec2 offset);
}
```

适用场景：

- JSON 导出
- 调试查看
- 复制粘贴选中子图
- 跨工具或跨宿主导入导出

### `IGraphPersistence`

图持久化接口，决定图最终如何保存到磁盘或宿主资产系统中。

```csharp
public interface IGraphPersistence
{
    void Save(Graph graph);
    Graph? Load();
    bool IsDirty { get; }
}
```

理解方式：

- `IGraphSerializer` 关注“转成什么格式”
- `IGraphPersistence` 关注“存到哪里、怎么存”

### `IUserDataSerializer`

业务数据序列化接口。框架本身不知道 `INodeData` 的具体结构，需要业务层提供这部分 JSON 序列化逻辑。

```csharp
public interface IUserDataSerializer
{
    string SerializeNodeData(INodeData data);
    INodeData? DeserializeNodeData(string typeId, string json);
}
```

当前代码中这里主要覆盖的是节点业务数据。

## 📚 接入建议

<a id="guide"></a>
### 最常见的接入分工

如果你是业务层使用者，通常只需要重点实现这些接口：

- `INodeTypeCatalog`
- `IConnectionPolicy` 或 `IConnectionValidator`
- `INodeContentRenderer`
- `IEdgeLabelRenderer`
- `IUserDataSerializer`

如果你是引擎宿主接入者，还需要准备：

- `IPlatformInput`
- `IEditContext`
- `ITextMeasurer`
- `IGraphPersistence`

### 当前最小 Unity 接入清单

| 项目 | 建议实现/使用 |
| --- | --- |
| 输入 | `UnityPlatformInput` |
| 编辑控件 | `UnityEditContext` |
| 文字测量 | `UnityTextMeasurer` |
| 渲染 | `UnityGraphRenderer` |
| 宿主窗口 | 自己的 `EditorWindow` |
| 业务扩展 | 节点类型目录、内容渲染器、连接策略、用户数据序列化器 |

### 一个判断原则

当你不确定某个能力应该放在哪层时，可以用这个简单标准：

- 会不会影响图的合法性：优先放 `Core` 侧策略接口
- 会不会影响节点里显示什么：优先放内容渲染器
- 会不会影响窗口事件如何转成编辑行为：优先放平台输入/宿主层
- 会不会影响保存格式或资产落地方式：优先放序列化/持久化层
